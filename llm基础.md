## 一、1\. 最大似然估计 MLE（模型原始目标：我要让真实句子出现概率最大）

### 1）自回归语言模型概率拆解（链式法则）

对于一句话 token 序列：$x_1,x_2,x_3...x_T$
联合概率拆解（概率论链式分解）：

- $P(x_t|x_{1:t-1})$：给定前面所有词，出现第$t$个真实词的概率（GPT 核心：上文预测下一词）

- $\theta$是 Transformer 全部参数（权重、偏置），$P$由$\theta$计算得到，$P=P(\cdot;\theta)$

### 2）最大似然 MLE 目标：最大化整条句子出现概率

我们训练目的：**调整参数**$\theta$**，让训练集里真实句子出现的概率尽可能大**

#### 为什么取对数$\log$？

1. 乘积$\prod$变求和$\sum$：$\log(ab)=\log a+\log b$，计算更简单，防止连乘数值下溢（小数越乘越小趋近 0，计算机精度爆炸）

2. $\log$是单调递增函数：最大化$P$ = 最大化$\log P$，优化方向不变

**MLE 最终优化目标：**$\boldsymbol{\max_\theta \sum_{t=1}^T \log P(x_t|x_{1:t-1};\theta)}$

> 关键：优化器 \(Adam/SGD\) 只能**最小化损失**，不能最大化，因此加负号→负对数似然 NLL 损失

### 补充小知识点：为什么不用 MSE（均方误差）？

1. 分类任务概率 0\~1，MSE 梯度平缓、收敛慢；交叉熵在预测偏差大时梯度陡峭，训练收敛更快

2. Softmax \+ 交叉熵数学上和 MLE 完全自洽，是离散分类最优损失

## 四、四、LLM（GPT 类自回归模型）预训练完整流程：从数据→前向→算损失→反向更新

### 步骤 1：数据预处理

1. 海量文本切分 token：`我爱中国 → [我,爱,中国]`，长度$T=3$

2. **输入 \- 标签错位拆分（自回归关键）**

    - 输入$input: [x_1,x_2,...,x_{T-1}] = [我,爱]$

    - 标签$label: [x_2,x_3,...,x_T]=[爱,中国]$
    逻辑：用前面所有 token，逐个预测后一个 token

### 步骤 2：模型前向传播

1. 输入 token 进 Embedding \+ 位置编码，逐层过 Transformer Decoder（GPT 全是 Decoder）

2. 每个位置$t$输出$V$维 logits（$V$= 词表大小），经过 Softmax 得到全词表概率分布$p$

### 步骤 3：损失计算（核心，用交叉熵 = 负对数似然）

逐个位置$t$拿真实标签$x_t$，计算该位置交叉熵$-\log P(真实词)$，全部 token 损失求和 / 求平均得到整句$\mathcal L$
$\mathcal L_{\text{pretrain}}=\frac1{T}\sum_{t=1}^T \text{CE}(pred_t,label_t)$

### 步骤 4：反向传播 \+ 参数更新

1. 自动微分求$\nabla_\theta \mathcal L$（损失对模型所有参数$\theta$的梯度）

2. AdamW 优化器沿梯度反方向更新权重：$\theta=\theta-\eta\cdot\nabla \mathcal L$（$\eta$学习率）

3. 循环遍历亿万文本，不断降低整体损失，直到模型能根据上文合理预测下一个词

## 五、关键总结（三者关系一句话）

1. **MLE（最大似然）：模型理论目标，最大化真实文本概率（最大化**$\sum\log P$**）**

2. **NLL（负对数似然）：数学变形，加负号转为最小化损失，对应你题目公式**

3. **交叉熵 CE：工程落地形式，one\-hot 分类下 NLL 的等价写法，框架代码直接调用 CrossEntropy**

4. **GPT 预训练本质：全程用交叉熵 / 负对数似然最小化，等价最大化训练数据似然**

## 拓展：BERT 和 GPT 损失区别（方便对照理解）

- GPT（自回归）：上文预测下文，损失是逐 token 自回归交叉熵

- BERT（掩码 MLM）：随机 mask 部分 token，预测被 mask 词，损失仅来自掩码位置交叉熵，不是全序列
